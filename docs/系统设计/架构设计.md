# 架构设计：Claude Code Manager

> 版本：v1.0
> 日期：2026-02-27

---

## 一、整体架构

Claude Code Manager 以**本地 Web 服务**形态运行：后端作为本地守护进程，前端为浏览器访问的 SPA。

```
┌─────────────────────────────────────────────────────┐
│                    浏览器 (SPA)                      │
│              Vue3 + TypeScript + Vite               │
│                   TailwindCSS                       │
└──────────────────┬──────────────────────────────────┘
                   │ HTTP / SSE
┌──────────────────▼──────────────────────────────────┐
│               本地后端服务                           │
│            Python + FastAPI                         │
│  ┌────────────┐  ┌─────────────┐  ┌──────────────┐ │
│  │ 项目管理   │  │  任务管理   │  │  执行引擎    │ │
│  └────────────┘  └─────────────┘  └──────┬───────┘ │
│  ┌────────────┐  ┌─────────────┐         │         │
│  │ 日志服务   │  │  文件写入   │         │         │
│  └────────────┘  └─────────────┘         │         │
└──────────────────────────────────────────┼──────────┘
                                           │ subprocess
                   ┌───────────────────────▼──────────┐
                   │        Claude Code CLI            │
                   │  (每个任务独立进程 + worktree)    │
                   └──────────────────────────────────┘
         ┌─────────────────────────────────────────────┐
         │            本地文件系统                      │
         │  data/projects.json                         │
         │  data/tasks/{project-id}.json               │
         │  data/logs/{task-id}.jsonl                  │
         │  {project-path}/docs/开发文档/任务清单.md   │
         │  {project-path}/docs/开发文档/经验总结.md   │
         │  {project-path}/docs/bug/*.md               │
         └─────────────────────────────────────────────┘
```

---

## 二、技术栈

| 层级 | 技术 |
| ---- | ---- |
| 前端框架 | Vue3 + TypeScript + Vite |
| 前端样式 | TailwindCSS |
| 后端框架 | Python 3.11+ + FastAPI |
| 数据存储 | JSON 文件（本地文件系统） |
| 实时推流 | SSE（Server-Sent Events） |
| 进程管理 | Python `subprocess`（asyncio 异步读写） |
| Git 操作 | 后端调用 `git` CLI 命令 |

## 三、系统分层

```
前端层          Vue3 SPA
                ├── views/        页面视图
                ├── components/   通用 UI 组件
                ├── composables/  业务逻辑组合函数
                ├── api/          HTTP 请求封装
                └── types/        TypeScript 类型定义

后端层          FastAPI 应用
                ├── routers/      HTTP 路由（RESTful + SSE）
                ├── services/     业务逻辑层
                ├── models/       Pydantic 数据模型
                └── core/         核心能力（进程、Git、文件）

数据层          本地 JSON 文件
                ├── projects.json
                └── tasks/{project-id}.json
                    logs/{task-id}.jsonl
```

## 四、后端模块划分

### 4.1 项目管理模块（ProjectService）

职责：管理已注册项目的元数据，读写 `data/projects.json`。

- 添加、列出项目
- 校验项目路径下是否存在 `docs/开发文档/任务清单.md`

### 4.2 任务解析模块（TaskParserService）

职责：将 `任务清单.md` 解析为结构化任务，写入 `data/tasks/{project-id}.json`。

- 按 Markdown 标题（`##` / `###`）分割任务边界
- 提取 `[feature]` / `[bug]` 类型标记
- 提取标题、描述、验收标准、所属模块
- 合并更新（保留已有任务状态，仅追加新任务）

### 4.3 任务管理模块（TaskService）

职责：任务的状态机管理，读写 `data/tasks/{project-id}.json`。

- 状态流转校验与更新
- 任务查询（按项目、按状态）
- 关联 Bug 报告路径、分支名、worktree 路径

### 4.4 执行引擎模块（ExecutionEngine）

职责：管理任务执行的完整生命周期，是系统核心。

子组件：

| 子组件 | 职责 |
| ------ | ---- |
| GitService | 创建分支、创建/删除 worktree、提交、推送 |
| PromptBuilder | 构造发送给 Claude Code 的 Prompt 文本 |
| ProcessManager | 启动/终止 Claude Code 进程，维护进程池 `{task_id: Process}` |
| OutputParser | 解析 stdout，识别 `SUCCESS` / `FAILURE:` 标记 |

执行流程：

```
触发执行
  → GitService.create_branch()
  → GitService.create_worktree()
  → PromptBuilder.build()（注入经验总结）
  → ProcessManager.start(task_id, worktree_path, prompt)
  → 更新任务状态 in_progress
  → OutputParser 异步监听 stdout
  → 识别结果标记
      SUCCESS  → 更新状态 pending_review
      FAILURE  → 更新状态 failed
               → FileWriter.append_experience()
```

### 4.5 日志服务模块（LogService）

职责：Claude Code 进程输出的实时推送与持久化。

- 将 stdout 每行写入 `data/logs/{task-id}.jsonl`（含时间戳）
- 通过 SSE 端点向订阅的前端推送每行日志
- 支持按 task_id 查询历史日志

### 4.6 文件写入模块（FileWriter）

职责：向被管理项目的文档文件追加内容。

- `append_experience(project_path, task_title, failure_reason)` → 追加至 `经验总结.md`
- `create_bug_report(project_path, task, failed_criteria)` → 在 `docs/bug/` 下创建 Bug 报告 `.md`
- `read_experience(project_path)` → 读取经验总结内容（供 PromptBuilder 注入）

## 五、前端模块划分

### 5.1 页面视图

| 页面 | 路由 | 组合函数 |

## 六、API 接口契约（概要）

### 项目

| 方法 | 路径 | 说明 |
| ---- | ---- | ---- |
| GET | `/api/projects` | 项目列表 |
| POST | `/api/projects` | 添加项目 |
| DELETE | `/api/projects/{id}` | 移除项目 |
| POST | `/api/projects/{id}/sync` | 重新解析任务清单.md |

### 任务

| 方法 | 路径 | 说明 |
| ---- | ---- | ---- |
| GET | `/api/projects/{id}/tasks` | 任务列表（支持状态过滤） |
| GET | `/api/tasks/{taskId}` | 任务详情 |
| POST | `/api/tasks/{taskId}/execute` | 触发执行 |
| POST | `/api/tasks/{taskId}/cancel` | 取消任务 |
| POST | `/api/tasks/{taskId}/reset` | 重置为 pending |
| POST | `/api/tasks/{taskId}/review` | 提交 review 结果（pass / fail + 失败的验收项） |

### 日志

| 方法 | 路径 | 说明 |
| ---- | ---- | ---- |
| GET | `/api/tasks/{taskId}/logs` | 历史日志列表 |
| GET | `/api/tasks/{taskId}/logs/stream` | SSE 实时日志流 |

### Bug

| 方法 | 路径 | 说明 |
| ---- | ---- | ---- |
| GET | `/api/projects/{id}/bugs` | Bug 报告列表 |

## 七、数据存储结构

### 7.1 projects.json

```
[
  {
    "id": "uuid",
    "name": "项目名称",
    "path": "/绝对路径",
    "createdAt": "ISO8601"
  }
]
```

### 7.2 tasks/{project-id}.json

```
[
  {
    "id": "uuid",
    "projectId": "uuid",
    "type": "feature | bug",
    "title": "任务标题",
    "description": "Markdown 原文",
    "module": "模块名",
    "status": "pending | in_progress | pending_review | completed | failed | cancelled",
    "acceptanceCriteria": ["条件1", "条件2"],
    "branch": "feat/xxx 或 null",
    "worktreePath": "绝对路径 或 null",
    "bugReportPath": "相对路径 或 null",
    "createdAt": "ISO8601",
    "updatedAt": "ISO8601"
  }
]
```

### 7.3 logs/{task-id}.jsonl

每行一条日志记录（JSON Lines 格式）：

```
{"ts": "ISO8601", "stream": "stdout | stderr", "line": "日志内容"}
```

## 八、关键流程说明

### 8.1 任务执行流程（执行引擎核心）

```
前端 POST /execute
  后端校验状态（必须为 pending）
  GitService: git checkout -b <branch> dev
  GitService: git worktree add .worktrees/<branch> <branch>
  FileWriter: read_experience → 读取经验总结
  PromptBuilder: 构造完整 Prompt
  ProcessManager: asyncio subprocess 启动 claude --print <prompt>
                  工作目录 = worktree 路径
  更新任务状态 → in_progress
  异步任务: 监听 stdout 每行
    → LogService: 写 jsonl + SSE 推送
    → OutputParser: 检测 SUCCESS / FAILURE:
        SUCCESS  → TaskService.update_status(pending_review)
        FAILURE  → TaskService.update_status(failed)
                 → FileWriter.append_experience()
```

### 8.2 SSE 日志流

```
前端 EventSource: GET /api/tasks/{id}/logs/stream
  后端维护 {task_id: [subscriber_queue]} 订阅表
  ProcessManager 每读到一行 stdout
    → 写 jsonl（持久化）
    → 向订阅队列 put(line)
  SSE endpoint 从队列 get() 后以 data: 格式发送
  进程结束 → 发送 event: done 关闭连接
```

### 8.3 Prompt 构造结构

```
[任务信息]
类型：feature / bug
标题：xxx
模块：xxx
描述：
  <完整 Markdown 描述>

[验收标准]
1. xxx
2. xxx

[工作区信息]
当前分支：feat/xxx
工作区路径：/path/.worktrees/feat/xxx

[已有经验总结]（若非空）
<经验总结.md 全文>

[执行要求]
完成后请在最后一行输出以下格式之一：
SUCCESS
FAILURE: <失败原因一行描述>
```
