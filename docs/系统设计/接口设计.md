# 接口设计：Claude Code Manager

> 版本：v1.0
> 日期：2026-02-27

---

## 一、通用约定

### 1.1 基础信息

| 项目 | 值 |
| ---- | -- |
| Base URL | `http://localhost:8765/api` |
| 数据格式 | `application/json` |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601（`2026-02-27T10:00:00+08:00`） |

### 1.2 统一响应结构

所有 REST 接口（非 SSE）均返回以下格式：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

`data` 为 `null` 表示无返回内容（如删除操作）。

### 1.3 错误码定义

| code | HTTP 状态码 | 说明 |
| ---- | ----------- | ---- |
| `0` | 200 | 成功 |
| `40001` | 400 | 请求参数错误 |
| `40401` | 404 | 资源不存在 |
| `40901` | 409 | 状态冲突（如任务状态不允许此操作） |
| `50001` | 500 | 服务器内部错误 |
| `50002` | 500 | Git 操作失败 |
| `50003` | 500 | Claude Code 进程启动失败 |

错误响应示例：

```json
{
  "code": 40901,
  "message": "任务当前状态为 in_progress，无法重复执行",
  "data": null
}
```

### 1.4 前端 TypeScript 类型定义

```typescript
// 通用响应包装
interface ApiResponse<T> {
  code: number
  message: string
  data: T
}

// 项目
interface Project {
  id: string
  name: string
  path: string
  createdAt: string
  taskStats: {
    total: number
    pending: number
    in_progress: number
    pending_review: number
    completed: number
    failed: number
    cancelled: number
  }
}

// 任务类型与状态
type TaskType = 'feature' | 'bug'
type TaskStatus =
  | 'pending'
  | 'in_progress'
  | 'pending_review'
  | 'completed'
  | 'failed'
  | 'cancelled'

// 任务
interface Task {
  id: string
  projectId: string
  type: TaskType
  title: string
  description: string
  module: string
  status: TaskStatus
  acceptanceCriteria: string[]
  branch: string | null
  worktreePath: string | null
  bugReportPath: string | null
  createdAt: string
  updatedAt: string
}

// 日志条目
interface LogEntry {
  ts: string
  stream: 'stdout' | 'stderr'
  line: string
}

// Bug 报告摘要
interface BugReport {
  filename: string
  relativePath: string
  taskId: string
  module: string
  feature: string
  description: string
  createdAt: string
}
```

## 二、项目接口

### 2.1 获取项目列表

```
GET /api/projects
```

**响应 data**：`Project[]`

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "id": "uuid",
      "name": "my-app",
      "path": "/Users/dev/my-app",
      "createdAt": "2026-02-27T10:00:00+08:00",
      "taskStats": {
        "total": 12,
        "pending": 5,
        "in_progress": 1,
        "pending_review": 2,
        "completed": 3,
        "failed": 1,
        "cancelled": 0
      }
    }
  ]
}
```

---

### 2.2 添加项目

```
POST /api/projects
```

**请求 Body**：

```json
{
  "name": "my-app",
  "path": "/Users/dev/my-app"
}
```

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| name | string | 是 | 项目显示名称 |
| path | string | 是 | 本地绝对路径 |

**校验规则**：
- `path` 目录必须存在
- `path` 下必须存在 `docs/开发文档/任务清单.md`
- `path` 不能与已有项目重复

**响应 data**：`Project`

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "uuid",
    "name": "my-app",
    "path": "/Users/dev/my-app",
    "createdAt": "2026-02-27T10:00:00+08:00",
    "taskStats": { "total": 0, "pending": 0, "in_progress": 0, "pending_review": 0, "completed": 0, "failed": 0, "cancelled": 0 }
  }
}
```

**错误**：

| code | 触发条件 |
| ---- | -------- |
| 40001 | path 不存在或缺少任务清单.md |
| 40901 | 该路径已被添加 |

---

### 2.3 移除项目

```
DELETE /api/projects/{projectId}
```

仅移除系统内的项目记录，不删除本地文件。

**响应 data**：`null`

---

### 2.4 同步任务清单

```
POST /api/projects/{projectId}/sync
```

重新读取 `docs/开发文档/任务清单.md`，解析并合并至任务数据。已有任务的状态不会被重置，仅追加新任务。

**响应 data**：同步结果摘要

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "added": 3,
    "unchanged": 9,
    "total": 12
  }
}
```

**错误**：

| code | 触发条件 |
| ---- | -------- |
| 40401 | 项目不存在 |
| 50001 | 任务清单.md 解析失败 |

## 三、任务接口

### 3.1 获取任务列表

```
GET /api/projects/{projectId}/tasks
```

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| status | string | 否 | 按状态过滤，多个用逗号分隔，如 `pending,failed` |
| type | string | 否 | 按类型过滤：`feature` 或 `bug` |

**响应 data**：`Task[]`，按 `updatedAt` 降序排列

---

### 3.2 获取任务详情

```
GET /api/tasks/{taskId}
```

**响应 data**：`Task`

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "uuid",
    "projectId": "uuid",
    "type": "feature",
    "title": "用户登录功能",
    "description": "## 需求描述\n实现用户名密码登录...",
    "module": "用户模块",
    "status": "pending",
    "acceptanceCriteria": [
      "输入正确用户名密码可登录成功",
      "输入错误密码显示错误提示"
    ],
    "branch": null,
    "worktreePath": null,
    "bugReportPath": null,
    "createdAt": "2026-02-27T10:00:00+08:00",
    "updatedAt": "2026-02-27T10:00:00+08:00"
  }
}
```

---

### 3.3 触发任务执行

```
POST /api/tasks/{taskId}/execute
```

无请求 Body。

触发后系统异步执行：创建分支 → 创建 worktree → 构造 Prompt → 启动 Claude Code 进程。接口立即返回，任务状态变为 `in_progress`。

**前置校验**：任务状态必须为 `pending`

**响应 data**：执行初始化信息

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "taskId": "uuid",
    "branch": "feat/uuid-用户登录功能",
    "worktreePath": "/Users/dev/my-app/.worktrees/feat/uuid-用户登录功能",
    "status": "in_progress"
  }
}
```

**错误**：

| code | 触发条件 |
| ---- | -------- |
| 40401 | 任务不存在 |
| 40901 | 任务状态不为 pending |
| 50002 | 分支或 worktree 创建失败 |
| 50003 | Claude Code 进程启动失败 |

---

### 3.4 取消任务

```
POST /api/tasks/{taskId}/cancel
```

无请求 Body。

若 Claude Code 进程仍在运行，后端同时终止该进程并清理 worktree。

**前置校验**：任务状态为 `pending`、`in_progress` 或 `pending_review`

**响应 data**：`null`

---

### 3.5 重置任务

```
POST /api/tasks/{taskId}/reset
```

无请求 Body。

将失败任务重置为 `pending`，清空 branch / worktreePath 字段，以便重新执行。

**前置校验**：任务状态必须为 `failed`

**响应 data**：`null`

---

### 3.6 提交 Review 结果

```
POST /api/tasks/{taskId}/review
```

**前置校验**：任务状态必须为 `pending_review`

**请求 Body**：

```json
{
  "result": "pass",
  "failedCriteria": []
}
```

```json
{
  "result": "fail",
  "failedCriteria": [
    "输入错误密码显示错误提示"
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| result | string | 是 | `pass` 或 `fail` |
| failedCriteria | string[] | result=fail 时必填 | 未通过的验收标准列表 |

**result = pass 时**，系统自动：

1. `git commit` + `git push`（在 worktree 内）
2. `git worktree remove`
3. 任务状态 → `completed`

**result = fail 时**，系统自动：

1. 在项目 `docs/bug/` 下生成 Bug 报告 `.md`
2. 创建新的 `bug` 类型任务，关联 Bug 报告路径
3. 任务状态 → `failed`

**响应 data**（pass）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "status": "completed",
    "branch": "feat/uuid-用户登录功能"
  }
}
```

**响应 data**（fail）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "status": "failed",
    "bugReportPath": "docs/bug/2026-02-27-用户模块-登录功能-错误提示未显示.md",
    "newBugTaskId": "uuid"
  }
}
```

**错误**：

| code | 触发条件 |
| ---- | -------- |
| 40001 | result=fail 但 failedCriteria 为空 |
| 40901 | 任务状态不为 pending_review |
| 50002 | git commit/push 失败 |

## 四、日志接口

### 4.1 获取历史日志

```
GET /api/tasks/{taskId}/logs
```

返回该任务的全部历史日志条目。

**响应 data**：`LogEntry[]`，按时间升序

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    { "ts": "2026-02-27T10:01:00+08:00", "stream": "stdout", "line": "Reading task description..." },
    { "ts": "2026-02-27T10:01:02+08:00", "stream": "stdout", "line": "Creating login controller..." },
    { "ts": "2026-02-27T10:05:30+08:00", "stream": "stdout", "line": "SUCCESS" }
  ]
}
```

---

### 4.2 实时日志流（SSE）

```
GET /api/tasks/{taskId}/logs/stream
```

**协议**：Server-Sent Events（`Content-Type: text/event-stream`）

前端使用 `EventSource` 订阅，后端将 Claude Code 进程 stdout 的每一行实时推送。

**事件类型**：

| event 字段 | 说明 |
| ---------- | ---- |
| `log` | 普通日志行（每行一条） |
| `done` | 进程结束，连接关闭 |
| `error` | 进程异常退出 |

**事件格式**：

```
event: log
data: {"ts":"2026-02-27T10:01:00+08:00","stream":"stdout","line":"Creating login controller..."}

event: log
data: {"ts":"2026-02-27T10:05:30+08:00","stream":"stdout","line":"SUCCESS"}

event: done
data: {"exitCode":0}
```

```
event: error
data: {"exitCode":1,"reason":"进程意外退出"}
```

**前端使用示例**：

```typescript
const es = new EventSource(`/api/tasks/${taskId}/logs/stream`)

es.addEventListener('log', (e) => {
  const entry: LogEntry = JSON.parse(e.data)
  logs.value.push(entry)
})

es.addEventListener('done', () => {
  es.close()
})

es.addEventListener('error', () => {
  es.close()
})
```

**说明**：
- 进程未启动时订阅，连接保持直到进程启动后开始推送
- 任务已结束（completed / failed / cancelled）时订阅，返回历史日志后立即发送 `done` 关闭

## 五、Bug 接口

### 5.1 获取 Bug 报告列表

```
GET /api/projects/{projectId}/bugs
```

扫描项目 `docs/bug/` 目录下的所有 `.md` 文件，解析文件头信息返回摘要列表。

**响应 data**：`BugReport[]`，按 `createdAt` 降序

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "filename": "2026-02-27-用户模块-登录功能-错误提示未显示.md",
      "relativePath": "docs/bug/2026-02-27-用户模块-登录功能-错误提示未显示.md",
      "taskId": "uuid",
      "module": "用户模块",
      "feature": "登录功能",
      "description": "错误提示未显示",
      "createdAt": "2026-02-27T10:05:00+08:00"
    }
  ]
}
```

| POST | `/api/tasks/{taskId}/cancel` | 取消任务 |
| POST | `/api/tasks/{taskId}/reset` | 重置任务为 pending |

| GET | `/api/tasks/{taskId}/logs/stream` | SSE 实时日志流 |
