> 任务编号：M1-T03

---

## 用户故事

作为 Claude Code Manager 后端开发者，我需要实现对 `data/` 目录下 JSON 文件的读写封装，以便为上层服务提供统一的数据访问接口。

---

## 功能描述

实现 `data/` 目录下 JSON 文件的原子读写操作，防止写入中断导致文件损坏，并支持并发写入时的文件锁机制。

### 3.1 核心函数

#### read_json(path)

- 功能：读取 JSON 文件并解析为 Python 对象/列表
- 文件不存在时：返回 `[]`（列表场景）或 `{}`（对象场景），不抛异常
- 文件损坏时：抛出包含解析错误信息的异常

#### write_json(path, data)

- 功能：将 Python 对象/列表写入 JSON 文件
- 实现原子写：先写入临时文件 `.tmp`，写入成功后重命名为目标文件
- 文件不存在时：自动创建
- 并发保护：同一文件写入时使用 `fcntl`（Linux/macOS）或 `msvcrt.locking`（Windows）文件锁，避免并发冲突
- 写入失败时：抛出包含错误信息的异常

## 验收标准

- [ ] 提供 `read_json(path)` / `write_json(path, data)` 工具函数
- [ ] `read_json` 文件不存在时返回空列表/空对象，不抛异常
- [ ] `write_json` 使用原子写（临时文件 + 重命名）
- [ ] 并发写入同一文件时加文件锁，不丢数据、不冲突
- [ ] 所有函数位于 `backend/app/core/file_utils.py` 中
- [ ] 支持跨平台（Windows / macOS / Linux）

---

## 边界情况

| 场景 | 预期行为 |
| ---- | -------- |
| 文件不存在（读） | 返回 `[]` 或 `{}`（根据上下文决定） |
| 父目录不存在（写） | 自动创建父目录 |
| JSON 格式损坏（读） | 抛出 `json.JSONDecodeError`，携带原始错误信息 |
| 并发写入同一文件 | 使用文件锁串行化，后写的操作需等待先写的操作完成 |
| 磁盘空间不足 | 写入临时文件失败时抛出 `IOError` |
