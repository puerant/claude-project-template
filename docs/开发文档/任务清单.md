## 执行流程说明

### 流程-6：任务执行标准流程

**模块**：开发规范
**状态**：已定义

**描述**

定义 Claude Code Manager 项目的标准任务执行流程，供后续所有功能任务参考。

**执行步骤**

1. 创建分支 feat/xxx（基于 master）
2. 创建 git worktree 独立工作区（.worktrees/feat/xxx）
3. 在 worktree 中执行 tasks.md 的原子任务（每步改一个文件）
4. 执行完成后验证验收标准
5. 验收通过 → 在 worktree 中提交代码（git commit）
6. 合并到主分支（master）并清理 worktree
7. 提交代码到远程（git push）

**注意事项**

- 每次执行前确保 master 分支是最新的
- worktree 路径位于项目根目录的 .worktrees/ 下
- 合并后及时清理 worktree，避免资源占用
- 功能完成后及时推送到远程仓库
